# Έλεγχος Ασφαλείας — call_logger (Ιούλιος 2026)

> Εγκάρσια σάρωση ~499 αρχείων Dart για ευπάθειες. Κάθε εύρημα είναι γραμμένο ώστε να
> μετατρέπεται σχεδόν αυτούσιο σε προτροπή Cursor: **αρχείο**, **γραμμές**, **σενάριο
> αποτυχίας**, **προτεινόμενη διόρθωση**, **στοχευμένα τεστ**.
>
> Πλαίσιο απειλής: desktop εφαρμογή για Windows 11, ενός/λίγων χρηστών IT support.
> Η βάση μπορεί να ζει σε **κοινό δικτυακό φάκελο (UNC)** και τα αντίγραφα ασφαλείας
> (`.zip`) μπορεί να **ανταλλάσσονται μεταξύ χρηστών**. Αυτά τα δύο σημεία ανεβάζουν τη
> σοβαρότητα ευρημάτων που αλλιώς θα ήταν αμελητέα σε αυστηρά τοπική εφαρμογή.
>
> Κατάταξη κατά σοβαρότητα (πιο κρίσιμο πρώτα).

**Σημεία που ελέγχθηκαν και βρέθηκαν ΕΝΤΑΞΕΙ** (για να μη σπαταληθεί χρόνος ξανά):
- **SQL injection**: όλα τα `rawQuery`/`rawUpdate` με τιμές χρήστη χρησιμοποιούν παραμέτρους `?`.
  Τα ονόματα πινάκων/στηλών που μπαίνουν με interpolation είναι είτε σταθερές (`AppConfig.*`)
  είτε περνούν από `_sqliteQuoteIdentifier`/`quoteId` (διπλασιασμός `"`). Δεν εντοπίστηκε injection.
- **Command injection**: όλες οι εκκινήσεις εργαλείων γίνονται με `Process.start(path, argsList,

  `{TARGET}`/`{EQUIPMENT_CODE}` περνούν ως ξεχωριστά argv και δεν εκτελούνται από κέλυφος.
- **CLI arguments** (`app_config.dart`): το `--profile` περνά από `_sanitizeProfileName`
  (regex `^[a-zA-Z0-9_-]+$`), σωστό.

## 1 — ΥΨΗΛΗ: Zip Slip / Path Traversal στην επαναφορά αντιγράφου

**Αρχείο:** `lib/features/database/services/database_backup_service.dart`
**Συνάρτηση:** `restoreFromBackupZip` — γραμμές **502–544** (ειδικά **510, 518, 527, 537**).
**Σημείο εισόδου:** ο χρήστης επιλέγει το `.zip` μέσω `FilePicker.pickFiles` στο
`lib/features/database/widgets/database_settings_panel.dart:1013` → `:1082`.

**Σενάριο αποτυχίας:**
Το όνομα κάθε εγγραφής μέσα στο zip χρησιμοποιείται ως εξής:
```
final rel = name.substring(mapsPrefix.length);          // π.χ. "maps/....."
final dest = File(p.join(mapsRoot, rel.replaceAll('/', p.separator)));
await dest.writeAsBytes(...);
```
Δεν υπάρχει έλεγχος ότι το `dest` παραμένει **μέσα** στο `mapsRoot`/`imagesRoot`/
`dictionariesRoot`/`lampDataBaseRoot`. Ένα κακόβουλο (ή αλλοιωμένο) zip με εγγραφή όπως
`maps/..\..\..\Users\<user>\AppData\Roaming\...\evil.bat` (ή `dictionaries/../../call_logger.exe`)
θα γράψει αρχείο **έξω** από τους φακέλους δεδομένων, δυνητικά αντικαθιστώντας εκτελέσιμα,
scripts εκκίνησης ή αρχεία ρυθμίσεων. Επειδή τα αντίγραφα ανταλλάσσονται, αυτό είναι
ρεαλιστικό διάνυσμα επίθεσης (RCE μέσω αντικατάστασης αρχείου στον φάκελο του exe, το οποίο
είναι εγγράψιμο σε portable εγκατάσταση).

Το ίδιο ισχύει και για την εγγραφή της βάσης (`.db`), αν και εκεί ο προορισμός είναι σταθερός
(`targetDatabasePath`), οπότε το κύριο πρόβλημα είναι στα maps/images/dictionaries/lamp.

**Προτεινόμενη διόρθωση:**
Πριν από κάθε `writeAsBytes`, κανονικοποίησε τον τελικό προορισμό και επιβεβαίωσε ότι είναι
κάτω από τη ρίζα. Πρόσθεσε βοηθητικό, π.χ.:
- `final resolved = p.normalize(p.join(root, rel.replaceAll('/', p.separator)));`
- απόρριψε αν `!p.isWithin(root, resolved) && resolved != p.normalize(root)`.
- Επιπλέον, απόρριψε ρητά εγγραφές με απόλυτη διαδρομή (`p.isAbsolute(rel)`) ή τμήματα `..`.
Κάνε το ίδιο και στους 4 κλάδους (maps/images/dictionaries/lamp). Κατέγραψε τις εγγραφές που
απορρίφθηκαν στο αποτέλεσμα (π.χ. πρόσθεσε `skippedUnsafeEntries` στο `DatabaseRestoreResult`)
ώστε να φανεί στον χρήστη.

**Στοχευμένα τεστ (δημιούργησε αρχείο αν δεν υπάρχει):**
```
flutter test test/features/database/services/database_backup_service_restore_test.dart
```
Τα τεστ πρέπει να: (α) φτιάχνουν εν μνήμη zip με εγγραφή `maps/../escape.txt` και να
επιβεβαιώνουν ότι **δεν** δημιουργείται αρχείο εκτός `mapsRoot`· (β) επιβεβαιώνουν ότι ένα
νόμιμο zip εξακολουθεί να επαναφέρεται κανονικά.

## 2 — ΜΕΣΑΙΑ-ΥΨΗΛΗ: Διαπιστευτήρια & API keys πάνω από καθαρό HTTP

**Αρχεία/γραμμές:**
- `lib/features/history/widgets/lansweeper/lansweeper_url_rules.dart:6–24` — τα
  `isApiEndpointUrl` και `isBrowserLaunchableUrl` δέχονται **και `http` και `https`**.
- `lib/core/services/lansweeper_helpdesk_login_probe.dart:34–41` — δέχεται `http` και κάνει
  **POST username+password** στη γραμμή `93–100`.
- `lib/core/services/lansweeper_sync_service.dart:88–99` — κανένας έλεγχος σχήματος· το
  **API key** μπαίνει στο query (`key=`) στη γραμμή `93–99` και στέλνεται όπως είναι.

**Σενάριο αποτυχίας:**
Αν ο χρήστης ρυθμίσει URL με `http://…`, το Lansweeper **API key** και τα **helpdesk
credentials** ταξιδεύουν χωρίς κρυπτογράφηση στο τοπικό δίκτυο. Οποιοσδήποτε στο ίδιο LAN
(ή man-in-the-middle) τα υποκλέπτει. Δεδομένου ότι μιλάμε για εσωτερικό δίκτυο IT support,
η υποκλοπή διαπιστευτηρίων helpdesk είναι σοβαρή.

**Προτεινόμενη διόρθωση:**
- Στο `isApiEndpointUrl` (και στο μονοπάτι της σύνδεσης/login) απαίτησε `scheme == 'https'`
  για τα endpoints που μεταφέρουν μυστικά (API key, credentials). Αν πρέπει να επιτραπεί
  `http` για εσωτερικούς servers χωρίς TLS, δείξε **ρητή προειδοποίηση** στο UI («τα
  διαπιστευτήρια θα σταλούν χωρίς κρυπτογράφηση») και ζήτησε επιβεβαίωση.
- Στο `lansweeper_sync_service.dart` πρόσθεσε έλεγχο: αν `baseUri.scheme != 'https'`, ρίξε
  `LansweeperSyncPrecheckException` ή προειδοποίησε.
- Το `isBrowserLaunchableUrl` (άνοιγμα σε browser) μπορεί να μείνει http/https, δεν μεταφέρει
  μυστικά μέσα από την εφαρμογή.

**Στοχευμένα τεστ:**
```
flutter test test/features/history/widgets/lansweeper/lansweeper_url_rules_test.dart
flutter test test/core/services/lansweeper_sync_service_test.dart
```
(Αν δεν υπάρχουν, δημιούργησέ τα: επιβεβαίωσε ότι `http://…/api.aspx` πλέον απορρίπτεται ή
σηματοδοτεί προειδοποίηση, ενώ `https://…/api.aspx` γίνεται δεκτό.)

## 3 — ΜΕΣΑΙΑ: Μυστικά αποθηκευμένα σε καθαρό κείμενο στη βάση

**Αρχεία/γραμμές:**
- `lib/core/database/settings_repository.dart:6,18–20` — κλειδιά `lansweeper_api_key`,
  `lansweeper_helpdesk_web_password`, `gemini_api_key`.
- `lib/core/services/settings_service_remote_lansweeper.dart:204–214` — αποθήκευση/ανάγνωση
  του **helpdesk web password** χωρίς καμία προστασία (`setLansweeperHelpdeskWebPassword`
  γράφει το raw value).

**Σενάριο αποτυχίας:**
Όλα τα μυστικά αποθηκεύονται ως καθαρό κείμενο στον πίνακα `app_settings` της SQLite. Αν η
βάση βρίσκεται σε **κοινό δικτυακό φάκελο (UNC)** — υποστηριζόμενο σενάριο, βλ.
`AppConfig.isUncDatabasePath` — ή αν το `.db` μπει σε αντίγραφο ασφαλείας που ανταλλάσσεται,
οποιοσδήποτε με πρόσβαση στο αρχείο διαβάζει API keys και τον κωδικό helpdesk με ένα απλό
`SELECT * FROM app_settings`. Δεν χρειάζεται καν η εφαρμογή.

**Προτεινόμενη διόρθωση (κατά σειρά προτίμησης):**
1. Κρυπτογράφησε τα μυστικά «at rest» με **Windows DPAPI** (`CryptProtectData`/
   `CryptUnprotectData` μέσω του υπάρχοντος `win32` package) δεμένα στον λογαριασμό χρήστη,
   πριν την αποθήκευση στο `app_settings`. Αποθήκευσε το κρυπτογράφημα (base64) και
   αποκρυπτογράφησε κατά την ανάγνωση.
2. Ελάχιστο μέτρο αν το (1) είναι πολύ: μην περιλαμβάνεις τα κλειδιά μυστικών στα αντίγραφα
   ασφαλείας/εξαγωγές, και τεκμηρίωσε ρητά τον κίνδυνο στο UI ρυθμίσεων.

**Στοχευμένα τεστ:**
```
flutter test test/core/services/settings_service_remote_lansweeper_test.dart
```
Επιβεβαίωσε round-trip: `set…` κρυπτογραφεί (η τιμή στη βάση ≠ plaintext) και `get…`
επιστρέφει το αρχικό.

## 4 — ΧΑΜΗΛΗ-ΜΕΣΑΙΑ: Gemini API key μέσα στο URL

**Αρχείο:** `lib/core/services/gemini_ticket_service.dart`
**Γραμμές:** `24–25` (`kDefaultGeminiEndpoint` με `?key=…`), `359–370` (`resolveEndpoint`
εισάγει το key στο template), `438–442` (list models με `key` στα queryParameters).

**Σενάριο αποτυχίας:**
Το API key τοποθετείται στο **query string** του URL. URLs καταλήγουν συχνά σε logs
(proxy, δίκτυο, exception traces). Επιπλέον το `endpointTemplate` είναι **επεξεργάσιμο από
τον χρήστη** και αποθηκευμένο· αν εμφανιστεί σε προεπισκόπηση ή μήνυμα σφάλματος μαζί με το
key, διαρρέει. (Σημείωση: επιβεβαιώθηκε ότι τα τρέχοντα μηνύματα `GeminiException`
χρησιμοποιούν το σώμα απόκρισης, όχι το URL — αλλά το μοτίβο παραμένει εύθραυστο.)

**Προτεινόμενη διόρθωση:**
Χρησιμοποίησε την κεφαλίδα `x-goog-api-key: <key>` αντί για `?key=` (το Gemini/Generative
Language API την υποστηρίζει) και **αφαίρεσε το key από το URL**. Έτσι το key δεν εμφανίζεται
ποτέ σε logs URL ούτε στο επεξεργάσιμο template. Πρόσθεσε ρητό guard: ποτέ μη συμπεριλάβεις το
πλήρες resolved endpoint σε μήνυμα σφάλματος/UI.

**Στοχευμένα τεστ:**
```
flutter test test/core/services/gemini_ticket_service_test.dart
```
Επιβεβαίωσε ότι το εκτελούμενο request φέρει κεφαλίδα `x-goog-api-key` και ότι το URI **δεν**
περιέχει το key.

## 5 — ΧΑΜΗΛΗ: Άνοιγμα αυθαίρετων διαδρομών/UNC από κείμενο κλήσης

**Αρχείο:** `lib/core/widgets/linkable_selectable_text.dart`
**Γραμμές:** `87–125` (`_openLink`, `_openFilesystemPath`).

**Σενάριο αποτυχίας:**
Το widget ανιχνεύει αυτόματα URL/UNC/τοπικές διαδρομές μέσα σε ελεύθερο κείμενο (σημειώσεις
κλήσης, λύσεις) και τα ανοίγει με `launchUrl(externalApplication)` ή με άνοιγμα του Explorer
σε **UNC path**. Αν κακόβουλο περιεχόμενο (π.χ. σημείωση που εισήχθη από άλλη πηγή/import)
περιέχει `\\attacker-host\share`, το πάτημα από τον χρήστη προκαλεί **SMB σύνδεση** προς τον
host του επιτιθέμενου → διαρροή NTLM hash / drive-by. Απαιτεί κλικ χρήστη, γι' αυτό ΧΑΜΗΛΗ.

**Προτεινόμενη διόρθωση:**
- Πριν το άνοιγμα, δείξε επιβεβαίωση για **UNC** (`\\`) και για μη-`http(s)` σχήματα,
  εμφανίζοντας τον πλήρη προορισμό.
- Για URL, περιόρισε το auto-open σε σχήματα `http`/`https`/`mailto` (τα υπόλοιπα μόνο με
  ρητή επιβεβαίωση). Το `_openLink` ήδη ελέγχει `uri.hasScheme` αλλά δεν φιλτράρει σχήμα.

**Στοχευμένα τεστ:**
```
flutter test test/core/utils/linkable_text_parser_test.dart
flutter test test/core/widgets/linkable_selectable_text_test.dart
```
(Αν δεν υπάρχει το δεύτερο, δημιούργησέ το: επιβεβαίωσε ότι UNC/μη-http προκαλεί διάλογο
επιβεβαίωσης αντί για άμεσο άνοιγμα.)

## 7 — ΧΑΜΗΛΗ: Δυναμικό PowerShell script σε `Process.run`

**Αρχείο:** `lib/core/database/lock_diagnostic_service.dart`
**Γραμμές:** `96–134` (`_runPowerShellFallback`), και `46–90` (`where`/`handle` με
`runInShell: true`).

**Σενάριο αποτυχίας:**
Δημιουργείται script PowerShell με **interpolation** των target διαδρομών βάσης και εκτελείται
με `Process.run('powershell', ['-Command', script], runInShell: true)`. Οι διαδρομές
διαφεύγονται (`\`→`\\`, `'`→`''`, lowercase), και **προέρχονται από ρυθμίσεις/config, όχι από
απομακρυσμένη είσοδο**, οπότε ο πρακτικός κίνδυνος είναι χαμηλός. Παραμένει όμως εύθραυστο
μοτίβο: αν κάποτε η διαδρομή βάσης γίνει επεξεργάσιμη/εισαγόμενη από τρίτους, ένα κατασκευασμένο
path θα μπορούσε να σπάσει τη διαφυγή ή να εισάγει εντολές.

**Προτεινόμενη διόρθωση (defense-in-depth):**
- Απόφυγε `-Command` με δυναμικό string· προτίμησε πέρασμα των targets ως **παραμέτρων** σε
  σταθερό script (π.χ. μέσω `$args` / περιβάλλοντος), ώστε να μη γίνεται ποτέ interpolation
  σε κώδικα.
- Επικύρωσε ότι κάθε target είναι απόλυτη διαδρομή αρχείου που υπάρχει, πριν το περάσεις.
- Θέσε `runInShell: false` όπου δεν χρειάζεται κέλυφος.

**Στοχευμένα τεστ:**
```
flutter test test/core/database/lock_diagnostic_service_test.dart
```
(Δύσκολο να τεσταριστεί end-to-end· τουλάχιστον unit-τεστ στη λογική κατασκευής/διαφυγής των
targets ώστε ένα path με `'` ή `;` να μη διαφεύγει ως κώδικας.)

## 8 — ΠΛΗΡΟΦΟΡΙΑΚΟ: Μη-διαφυγή wildcards σε `LIKE` (λειτουργικό, όχι ασφάλεια)

**Αρχεία/γραμμές:**
- `lib/core/database/calls_repository_dashboard.dart:12–14, 53–54, 61–62`
- `lib/core/database/audit_service.dart:485, 550`

**Σενάριο (λειτουργικό, όχι τρωτότητα):**
Οι τιμές αναζήτησης περνούν ως παράμετροι `?` (άρα **δεν** υπάρχει SQL injection), αλλά
τυλίγονται σε `'%$nq%'` χωρίς διαφυγή των χαρακτήρων `%`, `_`, `\`. Χρήστης που πληκτρολογεί
`%` ή `_` παίρνει λανθασμένα διευρυμένα αποτελέσματα. Καθαρά ζήτημα ορθότητας/UX, όχι ασφάλειας.

**Προτεινόμενη διόρθωση (προαιρετική):**
Πρόσθεσε βοηθητικό `escapeLike` (διαφυγή `\`, `%`, `_` με `\`) και `... LIKE ? ESCAPE '\'`
όπου χτίζεις pattern από είσοδο χρήστη. Χαμηλή προτεραιότητα.

**Στοχευμένα τεστ:**
```
flutter test test/core/database/calls_repository_dashboard_test.dart
```

## Προτεινόμενη σειρά διόρθωσης

1. **#1 (Zip Slip)** — μοναδικό εύρημα με δρόμο προς εγγραφή αρχείου εκτός sandbox· διόρθωσε πρώτο.
2. **#2 (HTTP plaintext)** και **#3 (μυστικά σε plaintext)** — μαζί, αφορούν την ίδια ροή Lansweeper/Gemini credentials.
3. **#4 (key σε URL)** — μικρή, καθαρή αλλαγή.
4. **#6 (νεκρό debug logger)** — τετριμμένη αφαίρεση, κάν' την οποτεδήποτε.
5. **#5, #7, #8** — defense-in-depth / λειτουργικά, χαμηλή προτεραιότητα.

> Υπενθύμιση για την εφαρμογή: για κάθε εύρημα, γράψε **πρώτα** τεστ που αναπαράγει το πρόβλημα
> και μετά τη διόρθωση, και τρέξε **μόνο** τα σχετικά αρχεία τεστ (βλ. εντολές ανά εύρημα) —
> ποτέ σκέτο `flutter test`.
