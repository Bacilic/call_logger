Εκαθάριση του lib\core\database\database_helper.dart μετά την ολοκλήρωση της εφαρμογής. Περιέχει πολλές εκδόσεις βάσης και πολλά άχριστα IF αφού η δομή της βάσης σταθεροποιήθηκε

Εκαθάριστη του lib\features\calls\screens\widgets\call_header_form.dart που έχει γίνει τεράστιο. Πρέπει να το σπάσουμε σε μικρότερα widgets

Στις εκκρεμότητες αφαίρεση του κουμπιού ολοκλήρωση για τις ολοκληρεμένες εκκρεμμότητες και επισύμανη του πότε ολοκληρώθηκε και διάρκεια σε μέρες-ώρες-λεπτά από την δημιουργία μέχρι την ολοκλήρωση

**Βάση Δεδωμέων**

 - Εύρεση σφαλμάτων: Ονόματα χωρίς τμήματα, ορφανά τηλέφωνα, κλπ
 - Αλλαγή μηνύματος στην Επεξεργασία Τμήματο όταν προσυθετουμε τηλέφωνο στο τμήμα που είναι ήδη στο τμήμα: Το τηλέφωνο ανήκει στο ονομα_τμήματος. Αφαίρεση από τον όνομα_χρήστη;

Από τις κλήσεις περνάει ίδιο τηλέφωνο σε διαφορετικά τμήματα

**Ιστορικό**
Αντί να κανονικοποιούμε (normalize) τα δεδομένα κατά την αναζήτηση, τα κανονικοποιούμε κατά την αποθήκευση. Θα δημιουργήσουμε μια κρυφή στήλη στη βάση (π.χ. search_index). Εκεί μέσα θα αποθηκεύουμε ένα τεράστιο, κανονικοποιημένο String (χωρίς τόνους, όλα μικρά) που θα περιέχει τον τίτλο, την περιγραφή, το τηλέφωνο κ.λπ

Επαναχρησημοποίηση του κωδικα για το Χ καθαρισμού σε κάθε πεδίο (δεν ισχύει γιατί το καθένα είναι ανά περίπτωση)

Όταν στην καρτέλα Ιστορικό και Εκκρεμμότητες δεν υπάρχουν δεδομένα ή προκύψει σφάλμα με τη βάση τότε όλα τα χειριστίρια να είναι ανενεργά

Όταν γίνονται προσθήκες στην βάση, αυτές να περνάνε αυτόματα στις εκρεμότητες (πχ νέος υπάλληλος, νεό τηλέφνωνο) ώστε να ελέγχονται στον κατάλογο

**Διωρθώσεις**

Από την ταυτόχρονη υποστήριξη παλιών string κλειδιών ("anydesk", "vnc") και νέων κλειδιών "<tool_id>", που το έχουμς ώστε να μη σπάσει υπάρχων δεδομένα. Πρέπει να γίνει μετάβαση μονο στο tool_id του πίνακα remote_tools και από την στήλη remote_params του πίνακα equipements
